# Dino Run — UI spec

How the Dino Run page should look, and where each visual fact lives in the
code. Treat this as the source of truth when restyling. Paths below are
relative to `ClaudeCode101/dino_run/`.

**Three files own the entire UI:**
- `templates/index.html` — page structure, the two overlays, copy.
- `static/style.css` — all page chrome and overlay styling (the canvas
  interior is *not* CSS — it is painted by JS).
- `static/game.js` — everything drawn *inside* the canvas (dino, obstacles,
  ground line, HUD), via the named color/size constants near the top.

## Reference screenshots

Ground truth, regenerated by `capture.py` (see bottom). Compare against
these after any UI change:

- `reference/running.png` — in play: airborne dino + HUD + ground line.
- `reference/gameover.png` — Game Over overlay over the frozen board.
- `reference/win.png` — You Win overlay.

## Layout

Single centered column on a near-white page. Top to bottom:

1. **`<h1>` title** — `🦖 Dino Run`, `2rem`, bold, dark `#333`.
2. **`.hint`** — `Press [Space] to jump. Survive 3 minutes to win!` in muted
   gray `#777`, with `Space` wrapped in a `<kbd>` chip.
3. **`.game-wrap`** (`position: relative; display: inline-block`) holding the
   `<canvas>` and the two absolutely-positioned overlays stacked on top.

Body: `text-align: center`, padding `24px 12px`, font stack
`"Segoe UI", system-ui, sans-serif`.

## Color palette

| Token | Hex | Used for |
|---|---|---|
| Page background | `#f7f7f7` | `body` |
| Canvas background | `#fbfbfb` | canvas fill (intentionally low-contrast vs page) |
| Ink / headings | `#333` | `h1`, HUD text, overlay `h2` |
| Muted text | `#777` / `#555` | hint / overlay paragraph |
| Line & buttons | `#535353` | canvas border, ground line, Replay button |
| Button hover | `#2f2f2f` | `.replay-btn:hover` |
| Dino | `#4caf50` (green) | player rectangle |
| Dino eye | `#fff` | 6×6 white square |
| Obstacle | `#c0392b` (red) | obstacle rectangles |

`kbd` chip: `#eee` fill, `#ccc` 1px border (2px bottom), `4px` radius,
monospace.

## The canvas

- Fixed **800 × 260** (`width`/`height` attributes — do not style its pixel
  size with CSS; that would stretch the coordinate space the JS draws in).
- `#fbfbfb` fill, `2px solid #535353` border, `6px` radius, `max-width: 100%`.

### Drawn inside the canvas (owned by `game.js`)

- **Ground line** — horizontal `#535353`, 2px, at `GROUND_Y = H - 40` (y≈220).
  Everything below it is "underground."
- **Dino** — green `#4caf50` rectangle, **44×47**, bottom-anchored at
  `x = 60`, with a small **white 6×6 eye** near its top-right. Jumps; never
  moves horizontally.
- **Obstacles** — red `#c0392b` rectangles, bottom-anchored on the ground,
  scrolling right→left. Width varies 18–33px, height 28–51px.
- **HUD** — `16px monospace`, ink `#333`:
  - top-left: `Time: 12.3s` (elapsed, left-aligned, x=12, y=24)
  - top-right: `Win in 2:59` (mm:ss countdown, right-aligned, x=W-12, y=24)

## Overlays (Game Over / You Win)

Both are full-board, **absolutely positioned `inset: 0`** inside
`.game-wrap`, flex-centered, over a **translucent white scrim**
`rgba(255,255,255,0.82)` with `6px` radius — the frozen board stays faintly
visible behind them. Default hidden via `.overlay.hidden { display: none }`;
exactly one shows at a time (driven by game state `running`/`gameover`/`won`).

**Overlay card** (centered):
- `<h2>` `1.8rem`: `💀 Game Over` or `🏆 You Win!`
- `<p>` muted `#555`:
  - Game Over — dynamic, e.g. `You survived 3.6s. Try again!`
  - Win — `You survived 3 minutes!`
- **Replay button** (`.replay-btn`): dark `#535353` fill, white text, `1rem`,
  padding `10px 24px`, `6px` radius, no border, pointer cursor; hover darkens
  to `#2f2f2f` over a `0.15s` transition. `Space` also replays.

## Design intent / guardrails

- **Minimal, monochrome chrome + two accent colors** (green player, red
  hazard). Keep new UI in the `#333`/`#535353`/`#777` grayscale unless it is
  the dino (green) or a hazard (red).
- The canvas interior is **canvas drawing, not DOM** — restyle it by editing
  the constants/`draw()` in `game.js`, not CSS.
- Overlays must stay **translucent** so the board reads behind them.
- Canvas pixel size is **logical coordinate space**; resize via the HTML
  `width`/`height` attributes (and matching JS), never CSS scaling.

## Regenerate the reference screenshots

The harness that produced the images above. Run after any UI change, then
open the PNGs and compare to this spec:

```bash
# from ClaudeCode101/dino_run/
pip install flask playwright
python -m playwright install chromium
python .claude/skills/dino-ui-spec/capture.py
```

Writes `running.png`, `gameover.png`, `win.png` to
`.claude/skills/dino-ui-spec/reference/`. It starts the Flask app itself,
drives the three states headlessly, and shortens the win timer to 2s in
flight (game.js on disk is untouched) so you don't wait 3 minutes.
